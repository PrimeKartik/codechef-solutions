## 📚 Key Learnings

**1. Make Triangle**
* The triangle inequality theorem allows for an `O(1)` solution; because at most *one* condition can fail at a time, you only need to identify the violating pair and calculate the exact delta `(largest - sum + 1)`.

**2. DIividing Candies**
* Combining a modulo divisibility condition `y % x == 0` with a running maximum check allows for simultaneous filtering and optimization in a single `O(N)` pass without storing the array.

**3. Final Element**
* Iterative adjacent XORing across an array maps perfectly to Pascal's Triangle modulo 2. This mathematical property allows an `O(N^2)` simulation to be completely bypassed in `O(N)` time using the bitwise AND trick `((N-1) & i) == i`.

**4. Finding All**
* When asked to evaluate outcomes across "all possible rearrangements" of an array, simulating permutations is an immediate Time Limit Exceeded (TLE). Counting the frequencies of the active elements and deducing reachable end-states via hardcoded conditional logic yields an `O(N)` solution.
