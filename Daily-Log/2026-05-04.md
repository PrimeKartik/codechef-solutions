## 📚 Key Learnings

**1. Minimum or Maximum**
* Validating prefix extremums doesn't require rebuilding the original array; you only need to track the expanding boundaries (`min` and `max`) and ensure every new element touches one of them.

**2. Distinct Codes**
* When counting short, limited-alphabet substrings (like 2-letter uppercase codes), using a 2D boolean/frequency array (`seen[26][26]`) is dramatically faster and more memory-efficient than using standard HashSets.

**3. Mode of Frequencies**
* Mapping a problem to a "frequency of frequencies" array naturally solves complex statistical modes, and iterating through this secondary array from index 1 upwards automatically handles "smallest value" tie-breaking.
